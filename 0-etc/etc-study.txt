■■■■■■■■■■
d-container1 에 진입해서 소스코드를 변경했다.
변경사항이 저장된 d-image1 도커 이미지를 만들려면?

1. container1 안에서 소스코드 변경.
   이때 컨테이너에서 exit 하지 말기
2. 터미널 하나 더 열고 docker commit d-container1 d-image1
3. 그 다음에 d-container1 에서 exit 하고 d-container1 삭제
4. d-image1 가지고 docker run 해서 컨테이너 띄워보면
변경사항 저장되어있음.

■■■■■■■■■■
docker run -v 으로 볼륨마운트한 소스코드가 있다.
컨테이너를 띄웠다.
컨테이너를 삭제하고 호스트에서 소스코드를 수정하고
다시 컨테이너를 띄웠다.
잉? 코드 수정 사항이 반영이 안돼있다
build 를 다시 해주고 docker run -v 해줘야한다.

■■■■■■■■■■
소스코드는 볼륨마운트해서 host 에서 수정할수 있는게 좋다
왜? host 에서 수정
-> docker build 해서 컨테이너에 반영
-> docker run 
이런식으로 하면 편하니까
바로 위 컨테이너 내의 코드 수정하는 방식은 번거롭다.
그리고 새로 빌드할때마다 그 수정사항은 사라진다
